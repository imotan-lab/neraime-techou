# うちどころ。

GitHub Pages で運用する静的スロット攻略サイトです。  
公開URL: https://imotan-lab.github.io/uchidokoro/

## 技術構成

- HTML / CSS（assets/css/practical.css）/ JavaScript
- 機種一覧データ：assets/data/machines.json
- 機種詳細記事データ：assets/data/machine-details/機種名.json
- Google Fonts（Orbitron）

## ファイル構成

- `index.html` トップページ
- `machine.html` 記事ページ（slugで機種切り替え）
- `checker.html` チェッカーページ（slugで機種切り替え）
- `setting.html` 設定判別ページ「ポチポチくん」（slugで機種切り替え）
- `assets/css/practical.css` 全ページ共通CSS
- `assets/data/machines.json` 機種一覧
- `assets/data/machine-details/` 機種別記事データ（JSON）

## 新機種追加の手順

1. `assets/data/machines.json` に機種データを追加
2. `assets/data/machine-details/機種名.json` を作成
3. `machines/機種名/index.html`・`checker.html` を追加（リダイレクト用）
4. `sitemap.xml` にURLを追加
5. GitHubにプッシュ

## 自動更新

GitHub Actions により毎日UTC18:00（日本時間3:00）にYouTube APIで人気ランキングを自動更新。

## 作業メモ（2026-03-20時点）

### 進め方

- 変更前に必ず現在のファイルを確認する
- いきなり実装せず 先に修正方針を出す
- ユーザーのOK後に実装する
- 差分は最小限にする
- 不要ファイルは追加しない
- 削除が必要な場合は事前確認する
- 変更後は `変更内容` `影響範囲` `確認ポイント` をまとめる
- アップ先ディレクトリとコミットサマリー文も毎回記載する

### 会話上の運用

- 基本はページ名で話す
- 聞かれていない改善提案は出さない
- 推測ではなく実記述ベースで答える
- 軽微な修正でも 先に修正方針を出す
- 関連する修正は なるべくまとめて入れる
- 公開確認はクリックできるURLで案内する
- ユーザーはスクショ確認が中心なので 機種名つきフルページスクショで判断できる前提で進める

### 記事ページの現在地

- 全機種分の記事データJSONは作成済み
- ただし北斗ページと比べると まだ情報量と精度に差がある
- 今は まずページとして成立させる段階を終えた状態
- 今後は 内容の精度確認 厚み追加 早見表と基本情報の役割整理 を行う

### 後でブラッシュアップする項目

- 北斗以外の記事ページは `狙い目早見表` と `基本情報` の役割がまだ重なっている
- 最終確認時は `たらればさんのnote` `ちょんぼりすた` `1Geki` `nana press` を突き合わせる
- 設定示唆まとめの色は 直感で強弱が分かりにくいので 後で改善候補
- 設定示唆まとめの右列バッジは 共通CSSで `1行優先`
- ただし長文だけは 機種ごとに個別確認して必要なら調整する

### 表示メモ

- 設定示唆まとめの表幅は ページ内で右列幅を統一する
- ポチポチくん導線の文言は `設定判別ツール ポチポチくん →`
- 表や図の文言は 箇条書き寄りなら句読点を減らす
- 詳細本文は 通常文として必要な `、` `。` を入れる
